# Roo Code 文档

Roo Code 是一套 AI 编码产品套件，旨在充分利用最先进的大语言模型，从根本上改变您创建软件的方式。它功能强大且可自定义，运行成本比其他替代方案更高，因为它使用具有实际文件系统访问权限、终端控制和多步骤工作流程的前沿模型。

Roo 的方法是用 token 换取质量。如果您想要获得目前可用的最佳和最有效的 AI 编码体验，这就是它。

Roo Code 有两种使用方式：
- [<LucideIcon name="Puzzle" style={{ width: "14px", display: "inline-block", verticalAlign: "middle" }} /> Roo Code VS Code 扩展](/getting-started/installing) 在您的 VS Code IDE 中本地运行，让您完全控制编码体验。非常适合深度或高度迭代的独立开发工作。高度可自定义，可融入您现有的工作流程。
- [<LucideIcon name="Cloud" style={{ width: "14px", display: "inline-block", verticalAlign: "middle" }} /> Roo Code Cloud Agents](/roo-code-cloud/cloud-agents) 是一个 24/7 工作的自主 AI 开发团队，作为一个团队协作，与您和其他同事合作完成各种任务。它在您工作的地方（Slack、Github 等）与您会合，允许大部分软件开发完全在 IDE 之外进行。

这两种选项高度互补，建立在相同的共享基础之上，实现同样出色的成果。

## Roo Code 能为您做什么

- 从自然语言描述和规范生成代码
- 重构和调试现有代码
- 编写和更新文档
- 回答关于您代码库的问题
- 自动化重复性任务
- 利用 MCP 服务器

## 成功的心态

最成功的 Roo 用户往往会接受以下理念：

- **利用模型无关性：** Roo 不是一个 LLM 模型，它需要一个 [LLM 提供商](/providers) 才能工作。但它兼容数十个提供商和数百个模型，因此您可以自由地实验、优化和切换，这是设计使然。在一个"最佳模型"每两周就变化的世界中，没有锁定。
- **不要吝啬 token：** 昂贵的最先进模型使用大量 token 几乎总是比使用少量 token 的便宜模型更胜一筹。注意您的预算，但要接受 token 比开发人员时间更便宜的事实。
- **信任角色：** 扩展的可自定义 Modes（架构师、代码等）和预定义的 Cloud Agent Types（规划师、编码员等）让模型保持在正轨上，专注于手头的任务，而不会超出其边界。根据您正在处理的任务依赖它们。
- **要有雄心：** Roo 在您更信任它时效果最好。开箱即用时，扩展会为每件事请求您的许可，但随着您变得更加舒适并启用 [自动批准](/features/auto-approving-actions)，您会看到它在没有卡住或做出错误决定的情况下独立工作很长一段时间。一旦准备就绪，将大型复杂项目交给 [Orchestrator](/basic-usage/using-modes#orchestrator-mode-aka-boomerang-mode)，它将在 Modes 之间协调任务。在 Cloud 中，我们为您处理所有这些。
