# 报告错误

当 Roo Code 遇到错误时，您可以导出详细的诊断信息并与支持团队分享。这有助于通过提供关于问题发生情况的完整上下文来更快地解决问题。

---

## 访问错误详情

当错误出现时，点击错误消息旁边的"详情"链接。这将打开一个包含诊断信息和导出选项的模态窗口。

<img src="/img/reporting-errors/reporting-errors-1.png" alt="点击错误消息上的详情链接以访问诊断选项" width="600" />

*详情链接显示了两个用于与支持团队共享错误信息的导出选项。*

---

## 选择正确的导出选项

错误详情模态窗口提供两个按钮，每个按钮服务于不同的需求：

1. **复制基本错误信息** - 将基本错误详情复制到您的剪贴板：
   - 时间戳和扩展版本
   - 正在使用的提供者和模型
   - 具体的错误消息

   **使用场景：** 当您想在 Discord 或简短的支持邮件中快速分享错误时。

2. **获取详细错误信息** - 打开完整的诊断报告，包含：
   - 基本信息中的所有内容
   - 完整的任务历史，显示所有消息和操作
   - 完整的配置和环境详情
   - 提供者特定的设置

   **使用场景：** 当支持团队需要重现问题或您正在提交 GitHub 问题时。

<img src="/img/reporting-errors/reporting-errors-2.png" alt="错误详情模态窗口，包含复制基本错误信息和获取详细错误信息按钮" width="600" />

## 与支持团队分享

复制错误信息后：

- **电子邮件：** 发送到 [support@roocode.com](mailto:support@roocode.com)
- **Discord：** 在 [#support channel](https://discord.gg/roocode) 中分享
- **GitHub Issues：** 在 [github.com/RooCodeInc/Roo-Code/issues](https://github.com/RooCodeInc/Roo-Code/issues) 提交

请包含对错误发生时您正在尝试执行的操作的简要描述。导出的诊断信息会处理技术细节。
