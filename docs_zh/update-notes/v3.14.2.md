# Roo Code 3.14.2 发布说明 (2025-04-24)

此版本引入了针对 Gemini 的提示缓存、用户对 OpenRouter 上特定模型缓存的控制、改进的终端输出处理，并增加了俄语语言支持。

---

## Gemini 2.5 缓存来了！

*   **Gemini 模型的提示缓存：** 当使用 [Requesty](/providers/requesty)、[Google Gemini](/providers/gemini) 或 [OpenRouter](/providers/openrouter) 提供商时，`Gemini 1.5 Flash`、`Gemini 2.0 Flash` 和 `Gemini 2.5 Pro Preview` 模型现在支持提示缓存（Vertex 提供商和 `Gemini 2.5 Flash Preview` 缓存即将推出！）
*   **手动缓存切换（仅限 Google Gemini 和 OpenRouter）：**
    *   针对 **[Google Gemini](/providers/gemini)** 和 **[OpenRouter](/providers/openrouter)** 提供商，新增了一个复选框，允许您手动为支持的 Gemini 2.5 模型启用提示缓存。
        <img src="/img/v3.14.2/v3.14.2.png" alt="Provider Settings 中的提示缓存复选框" width="600" />
    *   **为什么需要复选框？** 此设置作为临时解决方案，用于解决通过这两个提供商访问时 Google 缓存机制有时出现的响应延迟问题。对于它们，缓存默认是*不*启用的。
    *   **Requesty：** 通过 Requesty 支持的模型缓存仍然是自动的。
