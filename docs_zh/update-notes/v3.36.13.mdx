# Roo Code 3.36.13 版本说明 (2025-12-18)

此版本默认切换为原生工具协议，改进了 Gemini 模型的文件编辑功能，并增加了对 VS Code 语言模型提供者的原生工具支持。

<img src="/img/v3.36.13/v3.36.13.png" alt="Roo Code v3.36.13 Release" width="600" />

## 原生工具协议默认启用

支持原生工具调用的模型现在默认使用原生协议，而不是 XML。XML 协议仍可在提供者设置中使用 ([#10186](https://github.com/RooCodeInc/Roo-Code/pull/10186))。

## 生活质量改进

* **改进 Gemini 模型的文件编辑功能**：新的 edit_file 工具使 Gemini 模型在编辑文件时更加高效 ([#9983](https://github.com/RooCodeInc/Roo-Code/pull/9983))
* **VS Code LM 原生工具**：原生工具调用现在可与 VS Code 内置的 Copilot 模型配合使用 ([#10191](https://github.com/RooCodeInc/Roo-Code/pull/10191))

## Bug 修复

* **LiteLLM 工具协议下拉菜单**：原生/XML 协议选择器现在可正确显示在 LiteLLM 模型中 ([#10187](https://github.com/RooCodeInc/Roo-Code/pull/10187))
* **任务恢复**：更改原生工具调用设置后恢复任务时不再中断 ([#10192](https://github.com/RooCodeInc/Roo-Code/pull/10192))

## 其他改进

* **错误监控**：改进了对连续错误跟踪的处理 ([#10193](https://github.com/RooCodeInc/Roo-Code/pull/10193))
