# Roo Code 3.2 版本说明 (2025-02-27)

此版本周期主要专注于从 Roo Cline 到 Roo Code 的品牌重塑、自定义模式的引入，以及多项提供商更新和错误修复。

---

## 名称变更和自定义模式 (v3.2.0 - v3.2.2)

*   **从 Roo Cline 更名为 Roo Code：** 更名为 Roo Code 以更好地体现其身份。
*   **自定义模式：** 引入了为 Roo Code 创建自定义角色、定义自定义提示、为每种模式选择工具访问权限以及创建专业助手的功能。用户可以通过“提示”选项卡或输入“为  创建一个新模式”来开始使用。

---

## 提供商更新

*   **Bedrock 的 Llama 3.3 70B Instruct：** 将 Llama 3.3 70B Instruct 模型添加到 AWS Bedrock 提供商选项中。（感谢 @Premshay！）
*   **Gemini Flash Thinking 01-21 模型：** 添加了 `gemini flash thinking 01-21` 模型。（感谢 @monotykamary！）

---

## 错误修复和体验优化

*   **修复了打开自定义模式设置 JSON 的错误**
*   **还原了提供商密钥输入检查：** 将提供商密钥输入从 `onChange` 改回 `onInput`，以解决输入 API 密钥时的问题。（感谢 @samhvw8！）
*   **为 OpenAI 兼容提供商添加了 Azure 复选框：** 为使用 Azure 的 OpenAI 兼容提供商添加了一个显式复选框。（感谢 @samhvw8！）
*   **修复了 Glama 使用情况报告。**（感谢 @punkpeye！）
*   **修复了新配置文件的创建错误。**
*   **修复了内置模式角色定义覆盖的错误。**
*   **Gemini Flash Thinking 模型的视觉修复。**（感谢 @monotykamary！）
*   **差异工具使用控制：** 仅在设置中启用时才允许使用差异工具。
*   **修复了语言选择器错误。**
