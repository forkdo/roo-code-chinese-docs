# Roo Code 3.36 版本说明（综合）

Roo Code 3.36 引入了非破坏性上下文管理、新的调试和 UI 控件，以及持续的可靠性修复和提供商改进。

<img src="/img/v3.36.0/v3.36.0.png" alt="Roo Code v3.36 Release" width="600" />

## 非破坏性上下文管理

上下文压缩和滑动窗口截断现在会在内部保留原始消息，而不是删除它们 ([#9665](https://github.com/RooCodeInc/Roo-Code/pull/9665))。当您回退到较早的检查点时，完整的对话历史记录会自动恢复。

## GPT-5.1 Codex Max 支持

Roo Code 支持 GPT-5.1 Codex Max，OpenAI 的长期编码模型，包括 gpt-5.1 / gpt-5 / gpt-5-mini 变体的模型默认值 ([#9848](https://github.com/RooCodeInc/Roo-Code/pull/9848))。

## 浏览器截图保存

浏览器工具现在可以通过新的 `screenshot` 操作将截图保存到指定文件路径，以便在浏览器自动化任务期间捕获视觉状态 ([#9963](https://github.com/RooCodeInc/Roo-Code/pull/9963))。

## 超高推理强度

如果您将 gpt-5.1-codex-max 与 OpenAI 提供商一起使用，现在可以选择“超高”推理强度级别，以在复杂任务上获得最大推理深度 ([#9900](https://github.com/RooCodeInc/Roo-Code/pull/9900))。

## OpenRouter 原生工具默认

支持原生工具的 OpenRouter 模型现在默认使用原生工具调用，无需手动配置即可提高工具调用的可靠性 ([#9878](https://github.com/RooCodeInc/Roo-Code/pull/9878))。

## 错误详情模态框

悬停在错误行上可显示一个信息图标，点击后会打开一个包含完整错误详情和复制按钮的模态框 ([#9985](https://github.com/RooCodeInc/Roo-Code/pull/9985))。

## GPT-5.2 模型支持

GPT-5.2 在 OpenAI 提供商中可用，并设置为默认模型 ([#10024](https://github.com/RooCodeInc/Roo-Code/pull/10024))。

## Enter 键行为切换

您现在可以配置聊天输入中 Enter 键的行为，使其更好地适应多行提示和不同的输入方法 ([#10002](https://github.com/RooCodeInc/Roo-Code/pull/10002))。

## Gemini 3 Flash 预览模型

`gemini-3-flash-preview` 模型现在在 Roo Code Cloud 提供商、Google Gemini、GCP Vertex AI、Requesty 和 OpenRouter 提供商中可用。这是 Google 发布的最新模型（感谢贡献者！）（[#10151](https://github.com/RooCodeInc/Roo-Code/pull/10151)）。

## DeepSeek reasoner：工具使用期间的交错思考

DeepSeek 提供商的 `deepseek-reasoner` 模型现在支持“交错思考”和原生工具调用。在我们的内部评估中，工具调用成功率为 100%，扩展运行得分提高到 93.4%（感谢 zbww_！）（[#9969](https://github.com/RooCodeInc/Roo-Code/pull/9969)，[#10141](https://github.com/RooCodeInc/Roo-Code/pull/10141)）。

## 原生工具协议默认

支持原生工具调用的模型现在默认使用原生协议而不是 XML。XML 协议仍在提供商设置中可用 ([#10186](https://github.com/RooCodeInc/Roo-Code/pull/10186))。

## Vertex AI：Claude Sonnet 4.5 的 1M 上下文窗口

当您在 **Vertex AI** 上使用 **Claude Sonnet 4.5** 时，可以为支持的模型启用 **1M 上下文窗口** 选项 ([#10209](https://github.com/RooCodeInc/Roo-Code/pull/10209))。

## 聊天错误故障排除改进

聊天错误状态现在更容易理解出错原因，并在提交错误报告时更容易分享正确的详细信息：

* **更清晰的错误可见性**：错误行通过更明显的 **查看详情** 操作更一致地显示完整错误详情（包括状态代码） ([#10204](https://github.com/RooCodeInc/Roo-Code/pull/10204))
* **可下载的诊断信息**：您可以从聊天错误生成本地诊断文件（包括错误元数据和 API 对话历史记录），以便审查/编辑并与问题报告共享 ([#10188](https://github.com/RooCodeInc/Roo-Code/pull/10188))
