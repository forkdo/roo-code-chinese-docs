# Roo Code 3.12 版本说明（2025-04-16）

此版本引入了 xAI 提供商支持、改进了差异编辑功能、通过搜索功能增强了用户界面、增加了对 OpenAI 模型的支持，并包含各种可用性改进和错误修复。

*   添加了 [xAI 提供商](/providers/xai)，并公开了 OpenRouter 上 Grok 的推理努力选项。（感谢 Cline！）
*   添加了对 OpenAI `o3` 和 `4o-mini` 模型的支持。（感谢 PeterDaveHello！）

## 针对特定配置文件的差异设置

*   **针对特定配置文件的设置**：差异编辑配置现在基于每个配置文件工作，让您可以更好地控制不同提供商如何处理代码编辑。了解有关 [API 配置配置文件](/features/api-configuration-profiles) 的更多信息。

### 工作原理

* **多配置文件支持**：每个配置文件存储其自己的差异编辑首选项
* **灵活配置**：在配置文件之间切换可立即更改差异处理方式
* **针对特定提供商的控制**：为不同的代码提供商使用不同的差异策略
* **隔离设置**：一个配置文件中的更改不会影响其他配置文件

例如，您可以为某个提供商创建一个具有严格空白处理的配置文件，而为另一个提供商创建一个规则更宽松的配置文件。当您切换配置文件时，系统会自动应用适当的差异编辑配置。

## 键盘快捷键

### 用于输入接受的键盘快捷键

添加了 `roo.acceptInput` 命令，允许用户使用键盘快捷键而不是鼠标点击来接受输入或建议。（感谢 axkirillov！）此功能：

#### 主要优势

* **键盘驱动界面**：无需鼠标交互即可提交文本或选择主要建议按钮
* **改进的可访问性**：对于行动不便或鼠标使用不适的用户至关重要
* **Vim/Neovim 兼容性**：支持来自以键盘为中心环境的开发者的过渡
* **工作流效率**：减少开发任务期间在键盘和鼠标之间切换的上下文切换

有关详细设置和使用说明，请参阅我们新的 [键盘快捷键](/features/keyboard-shortcuts) 文档页面。

*   改进了预差异字符串规范化，以提高编辑可靠性，尤其是在对空白敏感的语言中。

*   改进了文件/文件夹上下文提及 UI，以提高可用性。（感谢 elianiva！）
*   添加了代码操作使用情况、提示增强使用情况和连续错误错误的遥测功能，以提高产品稳定性。

*   在任务标题中隐藏零成本值，以实现更简洁的 UI。（感谢 do-it！）
