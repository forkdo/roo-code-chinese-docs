# Roo Code 3.1 版本说明 (2025-02-27)

此发布周期引入了可自定义的角色定义和聊天模式说明、重新设计的提示词增强功能、新增 DeepSeek-R1 支持、实验性的统一差异算法、Mistral 提供商集成，以及针对配置文件和 UI 的若干错误修复。

---

## 主要功能与增强

*   **可自定义聊天模式提示词 (v3.1.0)：** 现在，您可以通过“提示词”选项卡或特定模式的 `.clinerules-mode` 文件自定义每个聊天模式（Code、Architect 和 Ask）的角色定义和说明。
*   **重新设计的提示词增强功能 (v3.1.0)：** “增强提示词”按钮现在可与任何提供商和 API 配置配合使用，允许完全自定义提示词。
*   **自动批准聊天栏 (v3.1.3)：** 新增了自动批准聊天栏，以实现更快速的交互。（感谢 Cline！）
*   **实验性统一差异算法 (v3.1.7)：** 可在设置中启用实验性的新统一差异算法。（感谢 @daniel-lxs！）

---

## 提供商更新

*   **DeepSeek-R1 支持 (v3.1.7)：** 新增对 DeepSeek-R1 模型的支持。（感谢 @philipnext！）
*   **Mistral 提供商 (v3.1.6)：** 集成了 Mistral 提供商。（感谢 Cline！）
*   **VS Code 语言模型（实验性）(v3.1.2)：** 新增对 VS Code 语言模型（包括 Copilot）的实验性支持。（感谢 @RaySinner / @julesmons！）
*   **Glama 的 PKCE 支持 (v3.1.2)：** 为 Glama 提供商实现了 PKCE 支持。（感谢 @punkpeye！）

---

## 生活质量改进与用户界面

*   **从聊天中复制 Markdown (v3.1.0)：** 新增了一个按钮，可直接从聊天中复制 Markdown 内容。
*   **模糊搜索改进 (v3.1.2)：** 增强了提及、历史记录和模型列表中的模糊搜索功能。（感谢 @samhvw8！）
*   **针对 Light+ 主题的视觉修复 (v3.1.1)：** 解决了 Light+ 主题下聊天输入和设置中的视觉故障。

---

## 错误修复

*   **配置文件修复 (v3.1.7, v3.1.6, v3.1.2)：** 解决了与配置文件保存和切换相关的多个错误，特别是针对 VSCode LM 的问题。（感谢 @samhvw8！）
*   **自动批准菜单修复 (v3.1.4 - v3.1.5)：** 修正了自动批准菜单中的错误。
*   **VS Code 语言模型集成错误 (v3.1.3)：** 修复了 VS Code 语言模型集成中的一个错误。

---

## 其他改进

*   **更新 VSCode 引擎要求 (v3.0.3)：** 将所需的 VSCode 引擎更新为 `^1.84.0`，以匹配 Cline。
*   **为 o1 系统提示词使用 'developer' 消息 (v3.1.2)。**
