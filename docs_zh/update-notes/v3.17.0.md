# Roo Code 3.17.0 版本发布说明 (2025-05-14)

此版本带来了 Gemini 隐式缓存、通过“何时使用”指南实现更智能的回旋镖编排、对“询问”模式和回旋镖准确性的改进、实验性的智能上下文压缩，以及更流畅的聊天体验。

---

## 通过 Gemini 缓存提升性能
使用支持缓存的 Gemini 模型的用户，将因利用隐式缓存而体验到性能提升和总体成本降低。

---

## 更智能的回旋镖编排
Roo Code 现在通过模式定义中新增的“何时使用”字段，为选择最适合您任务的模式提供了增强的指导。此字段允许模式创建者提供关于使用特定模式的最佳场景的具体说明。以前，或者如果某个模式未定义此字段，Roo 将依赖模式角色定义的第一句话来获取此指导。

*   **“何时使用”字段：** 自定义模式现在可以包含“何时使用”描述。Roo 会利用此文本，尤其是在[编排器（回旋镖）模式](/features/boomerang-tasks)中，在进行任务编排（例如，通过 [`new_task`](/advanced-usage/available-tools/new-task) 工具）或自动切换模式（例如，通过 [`switch_mode`](/advanced-usage/available-tools/switch-mode) 工具）时做出更明智的决策。
*   **改进的编排：** 通过利用“何时使用”字段，Roo 可以更好地理解每个模式的用途，从而实现更有效的任务委派和模式选择。
*   **回退到角色定义：** 如果某个模式的“何时使用”字段未填充，Roo 将使用该模式角色定义的第一句话作为默认摘要来指导其决策。

上图显示了“何时使用”描述的示例。默认情况下，标准[代码模式](/basic-usage/using-modes#code-mode-default)的此字段当前未填充。您可以在[自定义模式文档](/features/custom-modes)中了解有关配置此字段的更多信息。

## “询问”模式与回旋镖编排的改进

我们对 Roo 理解和响应您请求的方式进行了多项底层改进：

*   **“询问”模式改进：** “询问”模式已得到改进，能够提供更全面、更详细的解释，不那么急于建议或切换到实现代码（等待您更明确的提示），并且更频繁地使用 Mermaid 图表等图表进行澄清。
*   **更准确的回旋镖编排：** 用于启动新任务的 [`new_task`](/advanced-usage/available-tools/new-task) 工具（Roo 使用）的内部描述已简化，以便 AI 更好地理解。此内部改进确保了[回旋镖（编排器）功能](/features/boomerang-tasks)更可靠地触发，从而实现更流畅、更准确的自动化任务委派。

## 通过智能压缩实现更智能的上下文管理
我们引入了一项名为**智能上下文压缩** (`autoCondenseContext`) 的实验性功能，以主动管理冗长的对话历史记录并防止上下文丢失。

其工作原理如下：

*   **自动摘要：** 当对话接近其上下文窗口限制时（具体来说，当上下文窗口几乎已满时），Roo Code 现在会自动使用大型语言模型 (LLM) 来总结现有的对话历史记录。
*   **保留关键信息：** 目标是在保留最关键信息的同时减少历史记录的令牌计数，确保 LLM 对过去的交互有连贯的理解。这有助于避免旧消息被静默丢弃。
*   **检查点完整性：** 虽然为了持续的 LLM 调用而进行了摘要，但当您回退到旧检查点时，所有原始消息都会被保留。
*   **选择加入的实验性功能：** 默认情况下禁用，此功能可在“高级设置”的“实验性功能”下启用。请注意，用于摘要的 LLM 调用会产生成本，该成本当前未在 UI 的成本跟踪中显示。

有关此功能的更多详细信息，请参阅[智能上下文压缩文档](/features/intelligent-context-condensing)。

## 更流畅的聊天体验，更少的中断！ (感谢 Cline！)

我们对 Roo Code 体验进行了一些不错的调整，使其更加出色：

*   **即使 Roo 正在思考，也可以继续输入：** 现在，即使 Roo 正忙于处理您的当前请求，您也可以在聊天中输入下一条消息。无需再等待输入字段解锁——让您的想法自由流淌！
*   **查看更改时保持专注：** 我们改进了 Roo Code 在向您显示代码差异时处理光标焦点的方式。这意味着当 Roo 呈现更改以供审查时，对您的流程的干扰更少。

这些改进旨在让您的 Roo Code 交互感觉更流畅，干扰更少。
