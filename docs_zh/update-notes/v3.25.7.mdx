# Roo Code 3.25.7 版本说明 (2025-08-05)

此版本引入了 Groq 的 GPT-OSS 模型，增加了对 Claude Opus 4.1 的支持，带来了两个新的 AI 提供商（Z AI 和 Fireworks AI），并包含了许多生活质量改进。

## Groq GPT-OSS 模型

Groq 现在提供 OpenAI 的 GPT-OSS 模型，具有令人印象深刻的功能 ([#6732](https://github.com/RooCodeInc/Roo-Code/pull/6732))：

- **GPT-OSS-120b 和 GPT-OSS-20b**：具有 131K 上下文窗口的专家混合模型
- **高性能**：针对 Groq 基础设施上的快速推理进行了优化

这些模型为 Groq 已有的令人印象深刻的阵容带来了强大的开源替代方案。

## Z AI 提供商

Z AI（前身为 Zhipu AI）现已作为提供商提供（感谢 jues！） ([#6657](https://github.com/RooCodeInc/Roo-Code/pull/6657))：

- **GLM-4.5 系列模型**：可访问 GLM-4.5 和 GLM-4.5-Air 模型
- **双区域支持**：在国际和中国大陆端点之间选择
- **灵活配置**：通过区域选择轻松设置 API 密钥

> **📚 文档**：请参阅 [Z AI 提供商指南](/providers/zai) 了解设置说明。

## Claude Opus 4.1 支持

我们已在多个提供商中增加了对新 Claude Opus 4.1 模型的支持 ([#6728](https://github.com/RooCodeInc/Roo-Code/pull/6728))：

- **可用提供商**：Anthropic、Claude Code、Bedrock、Vertex AI 和 LiteLLM
- **增强功能**：8192 最大令牌、推理预算支持和提示缓存
- **定价**：$15/M 输入，$75/M 输出，$18.75/M 缓存写入，$1.5/M 缓存读取

注意：OpenRouter 对 Claude Opus 4.1 的支持尚不可用。
