# Roo Code 3.11.9 发布说明 (2025-04-07)

此补丁版本引入了按配置文件的速率限制、多规则文件支持、提供商更新、API 增加以及各种改进和错误修复。
---

## 自定义 Roo 指令 (`.roo/` 目录)

*   引入了在 `.roo/rules/`（适用于整个工作区）和 `.roo/rules-{modeSlug}/`（适用于特定模式）目录中放置多个指令文件的支持。Roo Code 会递归读取这些目录中的所有文件，按文件名的**字母顺序**将它们追加到系统提示中。
*   这比单个文件更好地组织和管理复杂的指令集。
*   如果相应的目录存在且包含文件，则此基于目录的方法现在优先于单个的 `.roorules` 或 `.roorules-{modeSlug}` 文件。（感谢 upamune！）
*   了解更多关于 [自定义指令](/features/custom-instructions) 的信息以及这如何适用于 [自定义模式](/features/custom-modes)。

---

## 按配置文件的速率限制

*   **速率限制**设置现在为每个 [API 配置配置文件](/features/api-configuration-profiles)单独配置。以前是全局设置，现在允许您根据正在使用的提供商、模型或特定配置文件设置来设置请求之间的不同最小延迟。（感谢 ross 和 olweraltuve！）
*   默认值仍为 0（禁用），这适合大多数用户。
*   在每个配置文件的选项中配置此设置。详情请参见 [API 配置配置文件](/features/api-configuration-profiles#creating-a-profile) 指南。关于使用跟踪的一般信息可在 [速率限制和成本](/advanced-usage/rate-limits-costs) 中获得。
