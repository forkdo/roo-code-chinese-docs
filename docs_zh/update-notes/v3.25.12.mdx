# Roo Code 3.25.12 版本发布说明（2025-08-12）

此版本为 Claude Sonnet 4 带来了巨大的上下文窗口升级、本地提供者可配置超时设置，以及对 OpenRouter 的最低推理支持。

## Claude Sonnet 4：100 万 Token 上下文窗口

我们已升级 Claude Sonnet 4 集成，以支持 [Anthropic 最新的 API 更新](https://www.anthropic.com/news/1m-context)，将上下文窗口从 200,000 token 增加到 100 万 token——提升了 5 倍 ([#7005](https://github.com/RooCodeInc/Roo-Code/pull/7005))：

- **超大上下文**：在单个对话中处理整个代码库、大量文档或多个大型文件
- **分级定价支持**：自动处理 Anthropic 针对扩展上下文使用的分级定价
- **UI 集成**：上下文窗口大小现在在模型信息视图中显示，并提供启用 100 万上下文功能的切换开关

这一重大升级使您能够处理更大的项目，并在不拆分对话的情况下跨大型代码库保持上下文。

> **📚 文档**：请参阅 [Anthropic 提供商指南](/providers/anthropic) 以获取设置和使用说明。

## 本地提供者可配置 API 超时

运行大型模型的本地 AI 提供商现在可以配置自定义超时设置，以防止过早断开连接 ([#6531](https://github.com/RooCodeInc/Roo-Code/pull/6531))：

- **灵活超时**：设置 0 到 3600 秒的超时时间（默认：600 秒）
- **提供商支持**：适用于 LM Studio、Ollama 和兼容 OpenAI 的提供商

在 VSCode 设置中配置：
```json
{
  "roo-cline.apiRequestTimeout": 1200  // 针对非常大的模型设置为 20 分钟
}
```

## OpenRouter 最低推理支持

OpenRouter 现在为兼容模型提供最低推理支持 ([#6998](https://github.com/RooCodeInc/Roo-Code/pull/6998))：

- **新的推理级别**：特定模型现在提供“最低”选项
- **UI 更新**：当适用时，思维预算界面会显示最低选项
- **优化性能**：更好地控制不同任务的推理强度

此新增功能提供了对模型推理的更精细控制，使您能够根据需求优化速度或深度。
