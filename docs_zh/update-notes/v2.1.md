# Roo Code 2.1 发布说明 (2025-02-27)

在引入 MCP 和主要版本 2.2 之前的这个发布周期中，重点进行了基础改进，包括实验性的差异编辑、OpenRouter 压缩、音效，以及使 Roo Cline（现为 Roo Code）能够在所有 VSCode 兼容编辑器中运行并与原始 Cline 并排工作。

---

## 主要功能和增强

*   **差异编辑（实验性引入 v2.1.12）：**
    *   初始实验性支持通过差异进行编辑。（感谢 @JoziGila！）
    *   后来在实验模式下切换到搜索/替换差异（v2.1.17）。
    *   修复了差异应用和 Windows 行尾（v2.1.14，v2.1.18）。
    *   明确了实验状态（v2.1.15）。
*   **OpenRouter 压缩（v2.1.11）：** 添加了对 OpenRouter 压缩的支持。（感谢 @lloydchang！）
*   **音效（v2.1.10）：** 加入了音效。（感谢 @HeavenOSK！）
*   **写入和命令的自动批准（v2.1.2）：** 对写入操作和命令执行的自动批准提供初始支持。
*   **使用 `.clinerules` 的自定义指令（v2.1.2 & v2.1.9）：** 支持 `.clinerules` 自定义指令，并在设置中添加了指导。
*   **浏览器操作自动批准（v2.1.3）：** 当 `alwaysAllowBrowser` 为 true 时，允许无需批准的浏览器操作。

---

## 兼容性和核心更新

*   **在所有 VSCode 兼容编辑器中运行（v2.1.6）：** Roo Cline（现为 Roo Code）已更新以在所有 VSCode 兼容编辑器中运行。
*   **与 Cline 并排运行（v2.1.4）：** 启用了 Roo Cline（现为 Roo Code）与原始 Cline 扩展并排运行。
*   **更新扩展图标和元数据（v2.1.7）。**

---

## 生活质量改进

*   **更大的文本区域和图像拖放（v2.1.21）：** 实现了更大的文本区域输入，并具有将图像拖入其中的功能。
*   **添加 Gemini 2.0（v2.1.20）。**
*   **从历史记录复制提示（v2.1.16）。**

---

## 错误修复

*   **音效设置（v2.1.13）：** 修复了音效不遵守设置的问题。
*   **浏览器操作批准修复（v2.1.5）。**
*   **差异 `write_to_file` 拒绝（v2.1.14）：** 如果启用了差异，会导致输出截断的 `write_to_file` 命令将被自动拒绝。
*   **Windows 行尾的差异编辑错误修复（v2.1.18）。**
*   **改进了差异编辑的错误处理（v2.1.19）。**
