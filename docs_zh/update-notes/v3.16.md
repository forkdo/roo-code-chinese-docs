# Roo Code 3.16 发布说明 (2025-05-12)

*这些发布说明包含 v3.16.x 中的所有改进。最后于 2025-05-12 使用 3.16.6 更新*

此版本为设置引入了垂直标签页导航，新增 API 提供商（Groq、Chutes、LiteLLM），可点击的代码引用，荷兰语本地化，稳定性增强（包括工具循环检测和改进的错误处理），UI 更新（包括临时的 Tailwind CSS 迁移及其回滚），更广泛的语言支持（包括 Elixir），以及提供商配置文件管理的改进，同时包含各种错误修复和杂项更新。

---

## Gemini 模型和缓存更新
- `gemini-2.5-pro-preview-05-06` 模型现在可用于 [Vertex](/providers/vertex) 和 [Google Gemini](/providers/gemini) 提供商。旧版 `gemini-2.5-pro-preview-03-25` 的用户将自动受益于这个较新的模型，因为之前的 ID 现在在 Google 后端映射到最新版本。无需更改配置。（感谢 zetaloop！）
- 提示缓存现在默认为 [Vertex](/providers/vertex) 和 [Google Gemini](/providers/gemini) 提供商上支持的 Gemini 模型启用，带来：
    - **重复查询的更快响应**：Gemini 记住之前的类似提示。
    - **减少 API 使用**：最小化冗余 API 调用。
    - **简化体验并可选择退出**：开箱即用，但可在设置中禁用。

## QOL 改进
### 新提供商：LiteLLM 集成
我们已引入对 [LiteLLM 提供商](/providers/litellm) 的支持，简化对各种语言模型的访问。此新集成提供：
*   **自动模型发现**：Roo Code 现在自动获取并列出 LiteLLM 服务器上可用的模型。这意味着用户不再需要在 Roo Code 中手动配置每个 LiteLLM 模型，简化了设置并使在 LiteLLM 提供的模型之间切换更容易。
*   **简化对 100+ LLM 的访问**：利用 LiteLLM 提供统一的 OpenAI 兼容 API 以支持各种底层模型的能力。

<img src="/img/litellm/litellm.png" alt="Roo Code LiteLLM 提供商设置" width="600" />

这个新提供商显著提高了通过 LiteLLM 使用多样化模型的便利性。有关设置 LiteLLM 的更多详细信息，请参阅 [LiteLLM 提供商文档](/providers/litellm)。

### Tool Loop 检测
我们已实施机制来检测和防止工具执行循环，增强稳定性和用户体验：
*   **防止无限循环**：系统现在识别工具何时可能陷入重复循环并智能干预，提示用户输入。
*   **改进的稳定性**：减少应用程序因意外工具循环而变得无响应或卡住的风险。

这确保了与扩展工具的更顺畅、更可靠且无挫折的交互。
*   **添加荷兰语本地化**：添加荷兰语支持，允许荷兰语用户以他们的母语使用扩展，提供更具包容性的体验。（感谢 Githubguy132010！）
*   **关于部分中的响应式页脚按钮**：修复了关于部分中页脚按钮的布局，确保它们在窄屏上正确换行，提供更好的移动体验和改进的可访问性。（感谢 ecmasx！）
*   改进自动批准切换的可访问性（感谢 Deon588！）
*   您现在可以直接为 Roo Code 设置切换 VSCode 的 `terminal.integrated.inheritEnv` 设置（感谢 KJ7LNW！）
*   在外部 API 中改进提供商配置文件管理
*   恢复"在外部 API 中改进提供商配置文件管理"
*   在预览中正确显示多行命令（感谢 KJ7LNW！）
*   在 read_file tool 中优雅处理不支持的语言错误（感谢 KJ7LNW！）
*   增强 select-dropdown 中的焦点样式并修复文档 URL（感谢 zhangtony239！）
*   在 UI 中正确处理模式名称溢出（感谢 elianiva！）
