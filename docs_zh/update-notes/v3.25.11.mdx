# Roo Code 3.25.11 版本说明 (2025-08-11)

此版本增强了 GPT-5 支持，添加了新的 IO Intelligence 提供商，并包含多项新功能、生活质量改进和错误修复。

## 增强的 GPT-5 支持

我们增强了 GPT-5 集成，使您能够利用更高级的功能，包括流式传输、多轮对话和高效的令牌管理。此版本还增加了对 OpenAI 新推出的 GPT-5 模型的支持，包括 `gpt-5-2025-08-07`、`gpt-5-mini-2025-08-07` 和 `gpt-5-nano-2025-08-07`。

> **📚 文档**：有关更多详细信息，请参阅 [OpenAI 提供商文档](/providers/openai)。

## 新的 IO Intelligence 提供商

我们添加了 IO Intelligence 作为一个新的提供商，通过统一的 API 为您提供对 Llama、DeepSeek、Qwen 和 Mistral 等广泛 AI 模型的访问 ([#6875](https://github.com/RooCodeInc/Roo-Code/pull/6875))。

> **📚 文档**：有关更多信息，请参阅 [IO Intelligence 提供商文档](/providers/io-intelligence)。

## Codex Mini 模型支持

我们在 OpenAI Native 提供商中增加了对 `codex-mini-latest` 模型的支持，使您能够直接利用其专门的代码相关功能（感谢 KJ7LNW！） ([#6931](https://github.com/RooCodeInc/Roo-Code/pull/6931))。

## 生活质量改进

* **`codebase_search` 工具**：明确了 `path` 参数是可选的，该工具默认搜索整个工作区 ([#6877](https://github.com/RooCodeInc/Roo-Code/pull/6877))。
* **`@roo-code/cloud` 链接**：通过允许从本地仓库直接链接 `@roo-code/cloud` 并支持热模块替换（HMR），改进了开发人员的工作流程 ([#6799](https://github.com/RooCodeInc/Roo-Code/pull/6799))。
* **聊天输入焦点**：现在，当从扩展的顶部菜单创建新聊天时，聊天输入框会自动获得焦点 ([#6689](https://github.com/RooCodeInc/Roo-Code/pull/6689))。
* **令牌使用量报告**：修复了令牌使用量和成本被低估的问题，提供了更准确的成本跟踪（感谢 chrarnoldus！） ([#6122](https://github.com/RooCodeInc/Roo-Code/pull/6122))。

## 错误修复

* **MCP 服务器启动**：修复了 MCP 服务器无法启动的问题，并移除了不必要的刷新通知 ([#6878](https://github.com/RooCodeInc/Roo-Code/pull/6878))。
* **工具重复检测器**：修复了将“错误和重复限制”设置为 1 时会错误地阻止第一次工具调用的问题（感谢 NaccOll！） ([#6836](https://github.com/RooCodeInc/Roo-Code/pull/6836))。
* **MCP 错误消息**：修复了 MCP 服务器错误消息显示原始翻译键的问题 ([#6821](https://github.com/RooCodeInc/Roo-Code/pull/6821))。
* **`apply_diff` 工具**：修复了处理复杂 XML 内容时导致 XML 解析错误的 bug ([#6811](https://github.com/RooCodeInc/Roo-Code/pull/6811))。
* **`max_tokens` 计算**：修复了对于具有非常大上下文窗口的模型，由于最大输出令牌计算不正确而导致请求失败的错误（感谢 markp018！） ([#6808](https://github.com/RooCodeInc/Roo-Code/pull/6808))。
* **滚动抖动**：修复了消息流式传输期间发生的滚动抖动问题，尤其是在代码块中 ([#6780](https://github.com/RooCodeInc/Roo-Code/pull/6780))。
* **MCP 服务器刷新**：修复了当保存不相关的设置时 MCP 服务器会不必要地刷新的问题 ([#6779](https://github.com/RooCodeInc/Roo-Code/pull/6779))。
* **AWS Bedrock 连接**：修复了使用 AWS Bedrock 与 LiteLLM 时的连接问题 ([#6778](https://github.com/RooCodeInc/Roo-Code/pull/6778))。

## 提供商更新

* **Fireworks**：增加了对四个新模型的支持：GLM-4.5、GLM-4.5-Air、gpt-oss-20b 和 gpt-oss-120b（感谢 alexfarlander！） ([#6784](https://github.com/RooCodeInc/Roo-Code/pull/6784))。
