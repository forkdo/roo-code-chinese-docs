# Roo Code 3.24.0 版本说明 (2025-07-26)

此版本引入了 Hugging Face 提供商（支持开源模型）、跨工具 AI 编码标准、增强的终端安全控制以及改进的诊断管理功能。

## 终端命令权限 UI

通过全新的交互式 UI，现在可以更轻松地管理终端命令权限（感谢 hannesrudolph！）（[#5798](https://github.com/RooCodeInc/Roo-Code/pull/5798)）：

- **可视化管控**：直接在聊天界面中查看和管理命令模式
- **模式建议**：根据命令获取智能模式推荐
- **切换控制**：轻松切换命令模式的允许/拒绝状态

## Hugging Face 提供商

我们新增了 Hugging Face 作为提供商支持，可直接访问数千个开源模型（感谢 TGlide、daniel-lxs！）（[#6134](https://github.com/RooCodeInc/Roo-Code/pull/6134)、[#6190](https://github.com/RooCodeInc/Roo-Code/pull/6190)、[#6228](https://github.com/RooCodeInc/Roo-Code/pull/6228)、[#6236](https://github.com/RooCodeInc/Roo-Code/pull/6236)）：

- **开源模型**：直接访问 Hugging Face 上庞大的社区模型库
- **灵活集成**：使用托管在 Hugging Face 基础设施上的模型
- **简易配置**：简单的设置流程，快速开始使用您偏好的模型和提供商

这为 Roo Code 打开了整个 Hugging Face 开源 AI 模型生态系统的大门。请参阅我们的 [Hugging Face 提供商文档](/providers/huggingface) 获取设置说明。

## 诊断控制

通过新增设置，现在可以控制上下文中显示的诊断消息数量（感谢 hannesrudolph！）（[#5582](https://github.com/RooCodeInc/Roo-Code/pull/5582)）：

- **限制错误和警告**：防止大量诊断信息填满模型上下文窗口
- **性能提升**：减少因处理过多诊断消息而导致的性能下降
- **遗留代码支持**：在开发过程中处理暂时存在大量错误的代码库时特别有用

此功能对于在使用包含大量诊断消息的大型或遗留代码库时保护上下文窗口至关重要。

## Agent Rules 标准支持

Roo Code 现在通过 AGENTS.md 文件支持 Agent Rules 标准（感谢 sgryphon！）（[#5969](https://github.com/RooCodeInc/Roo-Code/pull/5969)）：

- **跨工具兼容性**：在 Roo Code、Aider、Cline 及其他兼容 AI 工具之间共享自然语言指南
- **单一事实来源**：维护一套统一的编码标准、安全实践和工作流规则
- **自动检测**：Roo Code 自动查找并应用项目中的 AGENTS.md 文件

这解决了开发者使用多个 AI 工具时需要维护单独配置文件的分散问题。
