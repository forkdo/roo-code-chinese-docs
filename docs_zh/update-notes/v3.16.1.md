# Roo Code 3.16.1 版本发布说明 (2025-05-07)

此版本引入了对 LiteLLM 提供商的支持，提供更多 AI 后端选项；通过检测和防止工具执行循环来提高稳定性；新增荷兰语本地化；通过包含编辑器名称来增强遥测功能；UI 迁移至 Tailwind CSS 以提升一致性（在 v3.16.3 中暂时回滚）；修复了响应式页脚按钮；更新了评估默认设置；并升级了依赖项的最新版本，以提升安全性和性能。

---

## 新提供商：LiteLLM 集成
我们引入了对 [LiteLLM 提供商](/providers/litellm) 的支持，简化了对多种语言模型的访问。这一新集成提供了以下功能：
*   **自动模型发现**：Roo Code 会自动从您的 LiteLLM 服务器获取并列出可用模型。这意味着用户不再需要在 Roo Code 中手动配置每个 LiteLLM 模型，从而简化了设置流程，并更轻松地在 LiteLLM 提供的模型之间切换。
*   **简化对 100 多个 LLM 的访问**：利用 LiteLLM 为各种底层模型提供统一 OpenAI 兼容 API 的能力。

<img src="/img/litellm/litellm.png" alt="Roo Code LiteLLM 提供商设置" width="600" />

这一新提供商显著提升了通过 LiteLLM 使用多样化模型的便利性。有关设置 LiteLLM 的更多详细信息，请参阅 [LiteLLM 提供商文档](/providers/litellm)。

---

## 工具循环检测
我们实现了一种机制来检测和防止工具执行循环，从而增强稳定性和用户体验：
*   **防止无限循环**：系统现在能够识别工具是否可能陷入重复循环，并通过智能地提示用户输入来进行干预。
*   **提高稳定性**：降低了应用程序因意外工具循环而无响应或卡住的风险。

这确保了与扩展工具交互更加流畅、可靠且无挫败感。

---

## 生活质量改进
*   **新增荷兰语本地化**：增加了荷兰语语言支持，使荷兰语用户能够使用母语使用该扩展，从而获得更具包容性的体验。（感谢 Githubguy132010！）
*   **Tailwind CSS 迁移**：将 UI 迁移至 Tailwind CSS，以获得更精美和一致的界面。（注：此功能在 v3.16.3 中已回滚）
*   **关于部分中的响应式页脚按钮**：修复了关于部分中页脚按钮的布局，确保它们在窄屏幕上正确换行，从而提供更好的移动端体验和增强的可访问性。（感谢 ecmasx！）

---

## 其他改进
*   **遥测中的编辑器名称**：在遥测数据中添加了编辑器名称。这有助于了解哪些编辑器使用最广泛，并能够针对不同环境进行更有针对性的改进和支持。
*   **改进的评估默认设置和设置**：更新了评估默认设置并改进了设置流程，使评估环境更容易、更可靠地配置，并提供更实用的开箱即用设置。
*   **更新依赖项**：将依赖项更新至最新版本，以提升安全性和性能。
