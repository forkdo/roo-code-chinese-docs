# Roo Code 3.16.0 版本说明

*发布于 2025-05-06 的 Roo Code v3.16.0 版本说明。*

此版本引入了设置页面的垂直标签导航、新的 API 提供商（[Groq](/providers/groq) 和 [Chutes AI](/providers/chutes)）、可点击的代码引用，以及多项 UI/UX 改进，同时包含各种错误修复和其他改进。

---

## Gemini 模型和缓存更新
- `gemini-2.5-pro-preview-05-06` 模型现已可用于 [Vertex](/providers/vertex) 和 [Google Gemini](/providers/gemini) 提供商。使用旧版 `gemini-2.5-pro-preview-03-25` 的用户将自动受益于这个更新的模型，因为之前的 ID 现在在 Google 后端已指向最新版本。无需进行配置更改。（感谢 @zetaloop！）
- 提示缓存现已默认启用，适用于 [Vertex](/providers/vertex) 和 [Google Gemini](/providers/gemini) 提供商支持的 Gemini 模型，这将带来：
    - **重复查询的响应速度更快**：Gemini 会记住之前类似的提示。
    - **减少 API 使用量**：最大限度地减少冗余的 API 调用。
    - **开箱即用，可选择退出**：默认启用，但可在设置中禁用。

## 设置导航全面革新（感谢 @dlab-anton！）

设置界面已焕然一新，采用新的垂直标签布局，提供更高效、更直观的体验：
    - **一键访问**：通过设置视图中的新垂直标签，单击即可在设置部分之间导航。
    - **改进的布局和清晰度**：设置现在以清晰的垂直列表组织，以提高可见性。

## MCP 服务改进

    - MCP 服务器错误现在会被捕获并显示在新的“错误”标签页中（感谢 @robertheadley！）
    - 如果服务器连接正常，错误日志记录将不再破坏 MCP 功能（感谢 @ksze！）

---

## 聊天中的可点击代码引用（感谢 @KJ7LNW！）

导航 AI 响应中讨论的代码现在变得更加容易：
    - **可点击的代码和文件名**：AI 提到的任何 `代码` 或 `文件名.扩展名()` 现在都是可点击的链接。
    - **跳转到特定行**：链接会在编辑器中打开相关文件，并直接导航到引用的行号。
    - **简化代码探索**：从 AI 解释快速跳转到代码库中的确切位置。

## 持续的 UI/UX 改进（感谢 @elianiva！）

为获得更一致、更具视觉吸引力且更直观的体验，进行了常规 UI 改进：
    - **视觉统一设计**：在设置、提示交互和模式选择之间实现更一致的外观和感觉。
    - **改进的主题适应性**：在不同的 VS Code 主题之间实现更好的一致性。
    - **简化的交互**：整理了模式选择和提示增强区域等 UI 元素。
    - **代码块的现代化图标和工具提示**：代码块控件（复制、换行、展开/折叠）现在使用清晰的 Lucide 图标，并具有翻译后的工具提示，以提高可访问性。
