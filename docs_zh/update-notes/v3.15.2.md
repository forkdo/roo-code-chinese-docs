# Roo Code 3.15.2 发布说明（2025-05-02）

本次更新升级了 Boomerang Orchestrator 模式，改进了 Mermaid 图表的渲染和错误处理，提升了终端性能，新增了提供程序配置选项，并包含多项 UI 修复。

---

## Boomerang Orchestrator 模式

*   增强的 [Orchestrator (Boomerang) 模式](/features/boomerang-tasks) 严格编排任务执行。
*   不再读取、写入、执行命令或使用 MCP 服务器；这些任务已交由其他模式处理。
*   提供更可预测且更安全的任务自动化体验。

---

## 内联 Mermaid 渲染

*   通过更清晰、支持多语言的错误消息简化问题排查，不再直接显示无效的 Mermaid markdown 内容。
*   新增便捷按钮，可一键复制图表代码以便修复或分享。

    <img src="/img/v3.15.2/v3.15.2.png" alt="显示错误消息和复制代码按钮的 Mermaid 图表" width="600" />

---

## 终端性能

*   **提升终端性能与响应速度**
    *   显著优化了集成终端的运行速度，操作更加流畅。
    *   改进了命令中断的可靠性，尤其适用于产生大量输出的进程。

---

## 提供程序（Provider）

*   **新增 OpenAI API 基础 URL 自定义支持**
    *   允许受限区域用户或拥有自建基础设施的用户连接 OpenAI 服务。
    *   在设置中新增了相应配置选项（感谢 gongzhongqiang！）。

*   **为兼容 OpenAI 的提供程序新增自定义请求头支持**
    *   支持在设置中直接添加、编辑或删除自定义请求头，提升配置灵活性。
    *   简化了针对特定提供程序的配置需求（感谢 mrubens！）。

---

## 其他 UI 改进

*   **修复聊天输入框高度不稳定问题**
    *   修复了调整 Roo Code 窗口大小时聊天输入框高度异常跳动的问题。
    *   提供更稳定、更少干扰的聊天体验（感谢 zhangtony239！）。
*   **修正聊天界面内边距计算问题**
    *   修复了窗口调整大小时因内边距计算错误导致的间距异常或视觉错位。
    *   确保在任意窗口尺寸下界面更整洁、视觉更一致（感谢 zhangtony239！）。
*   **优化聊天中内联代码的样式**
    *   采用更小的字体和更柔和的边框，使技术讨论更清晰易读。
    *   减少视觉干扰（感谢 dicharkan！）。
*   **明确模式视图中工具组的显示状态**
    *   当某个模式未关联任何工具组时，明确显示“None”，避免歧义。
