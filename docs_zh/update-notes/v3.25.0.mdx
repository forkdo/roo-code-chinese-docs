# Roo Code 3.25.0 版本说明 (2025-07-29)

此版本为 Roo Code 带来了强大的新功能，包括用于工作流自动化的自定义斜杠命令、具有网络访问功能的增强型 Gemini 模型、全面的图像支持，以及无缝消息排队功能，让您的对话不受中断。

## 自定义斜杠命令

创建您自己的斜杠命令来自动化重复性工作流 ([#6263](https://github.com/RooCodeInc/Roo-Code/pull/6263), [#6286](https://github.com/RooCodeInc/Roo-Code/pull/6286), [#6333](https://github.com/RooCodeInc/Roo-Code/pull/6333), [#6336](https://github.com/RooCodeInc/Roo-Code/pull/6336), [#6327](https://github.com/RooCodeInc/Roo-Code/pull/6327))：

- **基于文件的命令**：在 `.roo/commands/` 中放置 markdown 文件即可立即创建自定义命令
- **管理界面**：用于创建、编辑和删除命令的新界面，内置模糊搜索功能
- **参数提示**：命令会在您输入时显示有关必需参数的有用提示
- **丰富描述**：添加元数据和描述，使命令具有自文档化特性

<img src="/img/slash-commands/slash-commands-4.png" alt="显示可用命令的斜杠命令菜单" width="600" />

将复杂的工作流转换为简单的命令，如 `/deploy` 或 `/review`，以加快开发速度。

> **📚 文档**：请参阅[斜杠命令指南](/features/slash-commands)获取详细使用说明。

## 消息排队

通过新的消息排队系统，在 Roo 处理您的请求时继续输入 ([#6167](https://github.com/RooCodeInc/Roo-Code/pull/6167))：

- **非阻塞输入**：即使在 Roo 处理之前的请求时，也可以输入和发送消息
- **顺序处理**：消息按发送顺序排队和处理
- **视觉反馈**：在界面中清晰显示排队的消息
- **保持上下文**：每条消息都保持对话的正确上下文

当您有多个快速问题或更正时，保持工作流程顺畅。

> **📚 文档**：请参阅[消息排队指南](/features/message-queueing)获取详细信息。

## read_file 工具的图像支持

[`read_file`](/advanced-usage/available-tools/read-file) 工具现在支持读取和分析图像（感谢 samhvw8！)([#5172](https://github.com/RooCodeInc/Roo-Code/pull/5172))：

- **多种格式**：支持 PNG、JPG、JPEG、GIF、WebP、SVG、BMP、ICO 和 TIFF
- **OCR 功能**：从截图和扫描文档中提取文本
- **批量处理**：读取文件夹中的多个图像并添加描述
- **简单集成**：就像读取文本文件一样工作 - 无需特殊配置

适用于分析 UI 模型、调试截图错误或从图像中提取代码。

## Gemini 工具：URL 上下文和 Google 搜索

Gemini 模型现在可以访问网页内容并执行 Google 搜索，以获得更准确、最新的响应（感谢 HahaBill！)([#5959](https://github.com/RooCodeInc/Roo-Code/pull/5959))：

- **URL 上下文**：直接分析网页、文档和在线资源
- **Google 搜索基础**：基于当前搜索结果获取经过事实核查的响应
- **用户控制**：根据您的隐私偏好启用或禁用网络功能
- **实时信息**：访问最新的文档和最佳实践

非常适合研究新库、验证解决方案或获取当前 API 信息。

> **📚 文档**：请参阅[Gemini 提供商指南](/providers/gemini)获取设置和使用说明。
