# Roo Code 3.23 版本说明

此版本将代码库索引功能提升至稳定功能，引入了强大的新待办清单来管理复杂任务，并包含关键的性能改进和错误修复！

## 代码库索引：始终开启，随时就绪

代码库索引已从实验性功能毕业，现已成为 Roo Code 的核心功能，可直接从聊天输入中使用。一旦配置完成，索引器将在后台自动运行，确保 Roo 始终对您的项目保持最新的语义理解。要免费开始使用，请参阅[代码库索引快速入门指南](/features/codebase-indexing#quick-start-guide)。

## 新功能：任务待办清单

此版本引入了一项新的待办清单功能，帮助您跟踪复杂任务。Roo Code 现在会为您的任务显示一个步骤清单，确保不会遗漏任何步骤。您可以直接在聊天界面中查看和管理待办清单。

<img src="/img/task-todo-list/task-todo-list-3.png" alt="Task Todo List overview showing interactive checklist in Roo Code" width="600" />

感谢 qdaxb 提供此功能！ ([#5182](https://github.com/RooCodeInc/Roo-Code/pull/5182), [#5472](https://github.com/RooCodeInc/Roo-Code/pull/5472), [#5473](https://github.com/RooCodeInc/Roo-Code/pull/5473), [#5482](https://github.com/RooCodeInc/Roo-Code/pull/5482))

## 新功能：命令拒绝列表

我们添加了自动拒绝工作流中不需要的命令的功能：

- **始终拒绝**：将命令标记为“始终拒绝”，以防止意外执行
- **节省时间**：无需重复手动拒绝相同的命令
- **工作流控制**：通过“始终拒绝”选项补充现有的自动批准功能

这为您在自动化工作流中的命令执行提供了更好的控制。

## Claude Code 的 Windows 支持

我们为 Claude Code 提供程序添加了原生 Windows 支持（感谢 SannidhyaSah、kwk9892！）（[#5615](https://github.com/RooCodeInc/Roo-Code/pull/5615)）：

- **修复 Windows 兼容性**：Claude Code 提供程序在 Windows 系统上不再卡住
- **改进输入处理**：使用基于 stdin 的输入，消除命令行长度限制
- **跨平台**：在 Windows、macOS 和 Linux 上稳定运行
