# Roo Code 3.23.13 版本说明 (2025-07-17)

此版本提高了代码库索引的可靠性，增强了 UI 的清晰度，并修复了一些重要错误。

## 代码库索引内存修复

我们解决了一个严重的内存泄漏问题，该问题在索引大型代码库时会导致崩溃（感谢 daniel-lxs、rxpjd、buck-0x、BenWilles！）：

- **降低内存使用量**：大型项目的内存消耗从约 500MB-1GB 降至仅 10-50MB
- **提高文件限制**：现在可以索引多达 50,000 个文件（之前为 3,000 个）
- **不再崩溃**：消除了索引过程中的内存不足错误

此修复使 Roo Code 在企业级代码库中更加可靠。

## Bug 修复

* **自定义模式名称**：修复了清除自定义模式名称会损坏 YAML 文件并使所有自定义模式消失的问题（感谢 daniel-lxs、kfxmvp！）（[#5794](https://github.com/RooCodeInc/Roo-Code/pull/5794)）
* **自动批准复选框**：解决了令人困惑的复选框状态，例如显示为选中但选择了“无”，或未选中但选择了选项（[#5602](https://github.com/RooCodeInc/Roo-Code/pull/5602)）
* **日期格式清晰度**：将日期格式更改为 ISO 8601，以防止 LLM 将 7/11/2025 等日期误解为 11 月 7 日而不是 7 月 11 日（感谢 chrarnoldus！）（[#5793](https://github.com/RooCodeInc/Roo-Code/pull/5793)）
* **设置保存问题**：修复了使用 OpenRouter 打开提供程序设置时需要放弃不存在的更改的错误（[#5779](https://github.com/RooCodeInc/Roo-Code/pull/5779)）
* **LiteLLM URL 处理**：修复了包含路径时的 baseURL 处理，确保请求发送到正确的端点（感谢 ChuKhaLi！）（[#5697](https://github.com/RooCodeInc/Roo-Code/pull/5697)）
* **项目分析**：修复了 list-files 工具，以确保在分析大型项目时可以看到完整的目录结构（感谢 qdaxb！）（[#5303](https://github.com/RooCodeInc/Roo-Code/pull/5303)）

## 生活质量改进

* **Ollama 超时**：将 API 超时从 10s/5s 增加到 60s/30s，以防止较慢的模型失败（感谢 daniel-lxs、danntee、vshvedov！）（[#5778](https://github.com/RooCodeInc/Roo-Code/pull/5778)）
* **Ollama UI**：更新为使用文本输入而不是下拉菜单来选择模型，与其他提供程序保持一致（感谢 daniel-lxs！）（[#5818](https://github.com/RooCodeInc/Roo-Code/pull/5818)）
* **设置组织**：将不常用的提供程序设置移动到“高级设置”下拉菜单中，以获得更清晰的 UI（[#5762](https://github.com/RooCodeInc/Roo-Code/pull/5762)）
* **错误控制**：添加了可配置的“错误和重复限制”设置，以控制在何时显示“Roo 遇到问题”对话框，并可以选择完全禁用它们（感谢 MuriloFP、anojndr！）（[#5752](https://github.com/RooCodeInc/Roo-Code/pull/5752)）
* **检查点效率**：从检查点中排除 Terraform 和 Terragrunt 缓存目录，将存储使用量减少了多达 10 倍（感谢 MuriloFP、ijin！）（[#5750](https://github.com/RooCodeInc/Roo-Code/pull/5750)）
* **消息编辑**：使用自定义模态框和改进的工作流彻底修改了消息编辑/删除界面（感谢 liwilliam2021！）（[#5538](https://github.com/RooCodeInc/Roo-Code/pull/5538)）

## 提供程序更新

* **Claude Code + Vertex AI**：添加了对使用 Claude Code 提供程序时 Vertex AI 模型名称格式化的支持（感谢 janaki-sasidhar！）（[#5728](https://github.com/RooCodeInc/Roo-Code/pull/5728)）
* **API 任务控制**：添加了通过 API 在启动任务时设置命令执行超时的功能（[#5825](https://github.com/RooCodeInc/Roo-Code/pull/5825)）
