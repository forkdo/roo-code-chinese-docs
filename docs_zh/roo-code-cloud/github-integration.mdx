# GitHub 集成

Roo Code Cloud 直接与 GitHub 集成，将您的自主 Cloud Agent 团队引入拉取请求工作流程。通过安装 Roo Code GitHub App，您可以让 agents 创建分支、自动审查代码并修复问题。

---

## 集成接触点

以下是 [Cloud Agents](/roo-code-cloud/cloud-agents) 与 GitHub 集成的方式：
- **代码创建：** Coder agent 通过 git 交付其工作。当您要求它创建/更改某些内容时，它会在其容器环境中进行更改，在本地提交，推送到远程仓库，并创建一个描述所有更改的 PR。您可以更改其设置，以创建草稿（而非常规）PR，或仅推送到远程分支而不创建 PR。
- **代码审查：** Reviewer agent 通过 Github PR 评论交付其工作。它将自动审查在其启用的仓库中新创建的 PR（包括 Coder 创建的！），通过添加评论。它还会审查推送到该分支的后续提交。您可以在其设置中控制哪些 PR 需要被审查的标准。
- **代码修复：** Fixer agent 通过推送到 Github 分支（和评论）交付其工作。当您从 PR 调用它时（见下文），它将阅读评论（由人类或 Cloud Agents 创建）并实施适当的修复，同时考虑您提供的任何额外说明。它会通过在其环境中检出 PR 分支、提交更改并将其推送到分支（并在完成后留下评论以便您知晓）来实现。

## 设置

首次创建 Cloud Agent 团队时，系统会提示您连接到 Github。如果您尚未完成此操作，请[立即进行](https://app.roocode.com/cloud-agents/setup)，并确保至少创建一个 PR Reviewer 和 PR Fixer agent。

这将引导您完成与 Github 的 OAuth 流程，在此过程中您将：

## 调用 agents

默认情况下，您拥有的任何 PR Reviewer agents 将根据您在[其设置](/roo-code-cloud/cloud-agents#settings)中定义的标准持续监控新的 PR。因此，当创建新的 PR 时，您将看到如下评论：

<img src="/img/integrations/gh-response-review.png" alt="Rooviewer agent 响应" width="400" />

您还可以通过如下方式调用 **@Roomote** 来让 PR Fixer agent 处理 PR 中的任何反馈：

<img src="/img/integrations/gh-mention.png" alt="提及 @roomote 以调用修复器" width="400" />

您也可以从它们自己的页面或顶部的“新建任务”按钮为 Reviewer 和 Fixer 分配任务。

它们的“新建任务”页面允许您粘贴相关 URL：

<img src="/img/integrations/gh-new-task.png" alt="为审查者新建任务" width="400" />

然后，您可以从 Web UI 跟踪任务，并在 PR 本身中查看结果。

:::info 始终是 @Roomote
请注意，您不能通过名称调用特定的 agents。当您使用 **@Roomote** 时，您是在提及 Roo Code GitHub App，该应用处理集成。然后，该应用将智能地为上下文调用适当的 agent。
:::
