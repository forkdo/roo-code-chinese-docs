Roo Code 3.25.7 版本说明 (2025-08-05)

此版本引入了 Groq 的 GPT-OSS 模型，增加了对 Claude Opus 4.1 的支持，带来了两个新的 AI 提供商（Z AI 和 Fireworks AI），并包含了许多生活质量改进。

Groq GPT-OSS 模型​

Groq 现在提供 OpenAI 的 GPT-OSS 模型，具有令人印象深刻的功能 (#6732)：

  • GPT-OSS-120b 和 GPT-OSS-20b：具有 131K 上下文窗口的专家混合模型
  • 高性能：针对 Groq 基础设施上的快速推理进行了优化

这些模型为 Groq 已有的令人印象深刻的阵容带来了强大的开源替代方案。

Z AI 提供商​

Z AI（前身为 Zhipu AI）现已作为提供商提供（感谢 jues！） (#6657)：

  • GLM-4.5 系列模型：可访问 GLM-4.5 和 GLM-4.5-Air 模型
  • 双区域支持：在国际和中国大陆端点之间选择
  • 灵活配置：通过区域选择轻松设置 API 密钥

📚 文档：请参阅 Z AI 提供商指南 了解设置说明。

Claude Opus 4.1 支持​

我们已在多个提供商中增加了对新 Claude Opus 4.1 模型的支持 (#6728)：

  • 可用提供商：Anthropic、Claude Code、Bedrock、Vertex AI 和 LiteLLM
  • 增强功能：8192 最大令牌、推理预算支持和提示缓存
  • 定价：$15/M 输入，$75/M 输出，$18.75/M 缓存写入，$1.5/M 缓存读取

注意：OpenRouter 对 Claude Opus 4.1 的支持尚不可用。

生活质量改进​

  • 多文件夹工作区支持：代码索引现在在多文件夹工作区中的所有文件夹中都能正确工作（感谢 NaccOll！） (#6204) - 了解更多
  • 检查点时机：检查点现在在文件更改之前保存，允许轻松撤销不需要的修改（感谢 NaccOll！） (#6359) - 了解更多
  • 重新设计的任务标题：更简洁、更直观的界面，具有改进的视觉层次结构（感谢 brunobergher！） (#6561)
  • 一致的检查点术语：移除了“初始检查点”术语以提高一致性 (#6643)
  • 响应式模式下拉菜单：模式选择下拉菜单现在可以随窗口正确调整大小（感谢 AyazKaan！） (#6422)
  • 性能提升：处理长 AI 响应时的性能显著提高（感谢 qdaxb！） (#5341)
  • 更简洁的命令批准 UI：简化界面仅显示唯一的命令模式 (#6623)
  • 智能待办事项列表提醒：待办事项列表提醒现在尊重配置设置（感谢 NaccOll！） (#6411) - 了解更多
  • 更简洁的任务历史记录：改进的任务历史记录显示，显示更多内容（3 行），预览中最多 5 个任务，以及简化的页脚 (#6687)
  • 内部架构：改进的事件处理以增强可扩展性 (#6606)

提供商更新​

  • Fireworks AI 提供商：新提供商提供流行的开源模型（如 Kimi 和 Qwen）的托管版本（感谢 ershang-fireworks！） (#6652)
  • Cerebras GPT-OSS-120b：在 Cerebras 提供商中添加了 OpenAI 的 GPT-OSS-120b 模型 - 免费使用，具有 64K 上下文和约 2800 令牌/秒 (#6734)

错误修复​

  • 模式名称验证：防止空模式名称导致 YAML 解析错误（感谢 kfxmvp！） (#5767)
  • 文本高亮对齐：修复了聊天输入区域高亮中的对齐错误（感谢 NaccOll！） (#6648)
  • MCP 服务器设置：正确尊重“启用 MCP 服务器创建”设置（感谢 characharm！） (#6613)
