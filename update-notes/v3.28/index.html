Roo Code 3.28 综合版本说明

此版本为 Roo Code Cloud 引入了 Supernova 模型，Task Sync 和 Roomote Control 功能可让您通过手机监控和控制任务，还包含 Roo Code Cloud 集成功能、增强的聊天编辑、对话保存改进以及多项错误修复。

Supernova 模型 - 免费使用​

我们已将全新的 roo/code-supernova 隐身模型添加到 Roo Code Cloud 提供商 - 促销期间完全免费 (#8175)：

  • 免费使用：无需 API 密钥，无需付费 - 通过 Roo Code Cloud 完全免费
  • 高级功能：强大的 AI 模型，支持图像功能，适用于多模态编码任务
  • 上下文窗口适中：200,000 个 token 的上下文窗口，最大输出 16,384 个 token
  • 隐身性能：体验 code-supernova 变体的尖端模型性能
  • 零配置：只需从 Roo Code Cloud 提供商中选择即可 - 无需任何配置

从 Roo 提供商中选择免费的 code-supernova 模型，即可零成本开始使用这一强大新功能。

文档：查看 Roo Code Router 指南 了解模型选择详情。

Task Sync 和 Roomote Control​

推出我们的新云连接功能，让您能够通过手机监控和控制长时间运行的任务 - 不再需要守在办公桌前！ (#7799, #7805, #7850)：

重要提示：Roo Code 仍然完全免费且开源。Task Sync 和 Roomote Control 是可选的补充服务，可将您的本地 VS Code 连接到云端 - 所有处理仍在您的 VS Code 实例中进行。

Task Sync (所有用户免费)：

  • 随时随地监控：离开办公桌时可通过手机查看长时间运行的任务
  • 实时更新：本地任务消息和进度的实时流式传输
  • 任务历史：您的任务会保存到云端，方便日后参考
  • 云端可见性：可通过任何浏览器查看您的 VS Code 任务

Roomote Control：

  • 远程继续任务：通过手机继续任务 - 响应提示、修复错误、批准操作
  • 完整聊天控制：如同在 IDE 中一样与聊天框交互
  • 启动/停止任务：随时随地启动新任务或终止正在运行的任务
  • 完全控制：从任何地方对本地 VS Code 进行双向完全控制

Task Sync 让您能够从任何设备监控本地开发环境。添加 Roomote Control 可获得完整的远程控制功能 - 无论您使用的是另一台电脑、平板电脑还是智能手机。

文档：查看 Task Sync 和 Roomote Control 指南。

GPT-5-Codex 登陆 OpenAI Native​

  • 处理仓库级上下文：借助 400k token 窗口，可在单个线程中保留多文件规范和长评论。
  • 更快重用提示并包含视觉内容：提示缓存和图像支持帮助您在无需重新上传上下文的情况下迭代 UI 修复。
  • 让模型自适应其努力程度：GPT-5-Codex 会自动平衡简单问题的快速响应与复杂构建的深入推理。

这为团队提供了无需额外配置的高容量 OpenAI 选项。#8260：

文档：查看 OpenAI 提供商指南 了解功能和设置指导。

生活质量改进​

  • 云账户管理：为已登录用户添加账户切换器和"创建团队账户"选项，更方便管理多个账户 (#8291)

  • 聊天图标尺寸：聊天界面图标在有限空间内现在保持一致的尺寸 (#8343)

  • 社交媒体分享：改进 OpenGraph 图像，在 Slack、Twitter 和 Facebook 上分享 Roo Code 链接时获得更好的预览卡片 (#8285)

  • 自动批准键盘快捷键：在编辑器任意位置使用 Cmd/Ctrl+Alt+A 切换批准状态，这样在审查更改时可以保持流畅的工作流程 (通过 #8214)

  • 点击编辑聊天消息：直接点击任何消息文本进行编辑，ESC 键取消，并改进了内边距一致性 (#7790)

  • 增强的推理显示：AI 的思维过程现在显示持久计时器，并以清晰的斜体文本显示推理内容 (#7752)

  • 更易于扫描的推理记录：在 UI 中的推理标题前添加清晰的分隔线，使长思维更易于浏览 (通过 #7868)

  • 手动认证 URL 输入：容器化环境中的用户在自动重定向失败时现在可以手动粘贴认证重定向 URL (#7805)

  • 活动模式居中：模式选择下拉菜单现在打开时会自动将活动模式居中显示 (#7883)

  • 保留第一条消息：包含斜杠命令或初始上下文的第一条消息现在在对话压缩期间会被保留，而不是被摘要替换 (#7910)

  • 检查点初始化通知：现在当检查点初始化失败时会收到清晰的通知，特别是在嵌套 Git 仓库中 (#7766)

  • 翻译覆盖审计：翻译检查器现在默认验证 package.nls 区域设置，以便在发布前捕获缺失的字符串 (通过 #8255)

  • 更小更微妙的自动批准 UI (感谢 brunobergher!) (#7894)

  • 登出时禁用 Roomote Control 以提高安全性 (#7976)

  • 为 cloudview 添加内边距以获得更好的视觉间距 (#7954)

  • 通过全局“启用”开关临时暂停自动批准功能，这样您就可以暂停自动批准而不会丢失每个开关的设置；UI 清晰地区分全局状态和每个开关的状态 (#8024)

  • 斜杠命令管理移至“设置”>“斜杠命令”，并可通过聊天弹出窗口上的齿轮快捷方式访问；弹出窗口现在仅用于选择，以便快速使用 (#7988)

  • 快速“添加到上下文”的键盘快捷键（macOS：Cmd+Y；Windows/Linux：Ctrl+Y） (#7908)

  • “添加图片”按钮移至文本区域内（右上角），“优化提示”按钮位于“发送”按钮上方，以减少鼠标移动并明确布局 (#7989)

  • Composer 操作按钮现在仅在存在文本输入时显示，从而减少视觉混乱和误点击 (#7987)

  • “新建任务”按钮图标从“+”更改为撰写/铅笔图标，以更好地匹配其用途并与 codicon 标准保持一致 (#7942)

  • 重新设计的消息流：享受更简洁、更易读的聊天界面，改进的视觉层次结构帮助您专注于重要内容 (#7985)

  • 响应式自动批准：自动批准下拉菜单现在可根据不同窗口大小自适应，采用智能的 1-2 列布局，工具提示显示所有已启用的操作，不会截断 (#8032)

  • 网络弹性：遥测数据现在在网络故障时自动重试，确保在连接问题期间不会丢失分析和诊断信息 (#7597)

  • 代码块默认换行：代码块现在默认换行，代码片段工具栏不再包含语言或换行切换按钮，从而确保代码片段在不同语言环境下都保持可读性（通过 #8194；#8208）

  • 重置会注销 Roo Code Cloud：重置现在还会注销 Roo Code Cloud，实现真正的全新开始；即使注销失败，重置仍会完成。 (#8312)

  • 衡量升级销售消息的参与度：可关闭升级销售消息的遥测功能跟踪关闭与点击（仅 upsellId），以调整消息传递并减少干扰；对用户没有可见的行为更改。 (#8309)

  • 欢迎屏幕 Roo 提供商实验：在欢迎屏幕上显示 Roo 提供商（本地化，带点击遥测），以加快设置速度。 (#8317)

  • 更简洁的提示（无“思考”标签）：移除 <thinking> 标签以获得更简洁的输出、更少的 token 和更好的模型兼容性；保留纯语言规则以确认工具成功。 (#8319)

  • 更简单的 Requesty 模型刷新：移除刷新 Requesty 模型列表时的不必要警告，这样您就可以获取新策略而无需离开屏幕。 (#7710)

  • Bedrock 1M 上下文复选框：添加复选框以启用 Bedrock 上 Claude Sonnet 4 和 4.5 的 1M 上下文窗口支持，为大型项目提供更多灵活性 (#8384)

  • 在云任务历史记录中包含推理消息，以便您在运行后查看完整的思考过程 (#8401)

  • 在扩展桥中识别云任务，以改进诊断、日志记录和未来的 UI 行为 (#8539)

  • 在遥测中添加父任务 ID，以提高可追溯性 (#8532)

  • 澄清 zh-TW “运行命令”标签，使其与工具提示匹配并减少混淆（感谢 PeterDaveHello！） (#8631)

  • 图片生成模型选择更新：默认为 Gemini 2.5 Flash Image；添加 OpenAI GPT-5 Image 和 GPT-5 Image Mini；更清晰的设置下拉菜单（感谢 chrarnoldus！） (#8698)
